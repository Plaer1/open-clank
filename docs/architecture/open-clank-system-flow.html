Open-Clank: Odysseus, MiMo Code, Frankenmemory, and the custom glue

Current architecture map of the hybrid stack: Odysseus remains the app shell, data plane, UI, auth, and owner-facing product. MiMo Code is the child agent brain. Open-Clank glue binds them through ACP, shared memory, permission UX, and repo governance.

Odysseus stays the product shell

  • Browser talks to FastAPI routes, not MiMo directly.
  • Odysseus owns sessions, auth, app.db, UI streams, and product tools.
  • Agent mode swaps the native loop for MimoSupervisor.bridge.run_turn().

MiMo becomes the agent brain

  • bin/mimo acp runs as a supervised child process.
  • ACP stdio carries prompts, model config, permission callbacks, and session updates.
  • The inner MiMo repo is clean on branch updated; outer repo still needs C3 submodule conversion.

Frankenmemory is the shared memory plane

  • Odysseus pre-turn recall and MiMo capture both call fm-mcp.
  • FM_DB_PATH converges all spawners on one SQLite DB.
  • Graph extraction, graph walk, groom, and code_index ride the same MCP surface.

Custom glue is the real integration

  • ACPBridge translates MiMo updates into Odysseus SSE.
  • PermissionHandler turns MiMo permission pauses into clickable Odysseus cards.
  • lifetools_server.py lets MiMo operate Odysseus-native tools without copying the app.
