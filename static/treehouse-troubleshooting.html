OpenClank Troubleshooting Guide

Common issues and their solutions. Organized by severity.

Startup & Connectivity

App won't start — "ModuleNotFoundError: No module named 'sqlalchemy'"

Python environment is missing dependencies.

Fix: pip install -r requirements.txt or uv pip install -r requirements.txt

Port already in use

Another process is using the configured port.

Fix: Check lsof -i :PORT or change the port in settings.json or via OPENCLANK_PORT env var.

Health endpoint returns degraded status

A dependent service (embedding, vector DB, calendar sync) is unavailable.

Fix: Visit the Diagnostics view to identify which service is degraded. Most optional services degrade gracefully — the app still works.

Browser shows "connection refused"

The server isn't running or the port is wrong.

Fix: Check terminal output for startup errors. Verify the URL matches the configured port.

LLM & Providers

"No API key configured for provider X"

The selected provider has no API key set.

Fix: Open Settings → Providers → enter your API key. Or set the env var (e.g., OPENAI_API_KEY).

Model returns empty response

Provider may be rate-limited or the model is unavailable.

Fix: Check provider status. Try a different model. Check the Diagnostics view for errors.

Tool calls fail with "tool not found"

The tool isn't enabled in permissions or the MCP server isn't connected.

Fix: Check tool permissions in Settings. For MCP tools, verify the server is running and connected.

Memory & Documents

Memory search returns nothing

Memory files may not exist yet, or the query is too generic.

Fix: Use specific terms (function names, file paths, task IDs). Check that memory files exist at ~/.claude/projects/.

Copal documents missing after restart

Database may not have been properly persisted.

Fix: Check that frankenmemory.db exists and is non-empty. Run the integrity check via Diagnostics.

TreeHouse

"TreeHouse state is corrupt"

The treehouse-state.json file has invalid JSON or schema.

Fix: Check ~/.odysseus/.copal/treehouse-state.json for valid JSON. If corrupted, restore from backup or create a fresh state.

"TreeHouse state changed in another tab" (stale revision)

Two tabs tried to write simultaneously.

Fix: Refresh the tab that has the stale revision. The optimistic concurrency check prevents data loss.

Activity completion blocked by prerequisites

The skill prerequisite or module-order gate is not met.

Fix: Complete the required prerequisite activities first. Check the skill prerequisite tree and module order.

Performance

Slow response times

Large context window or many tools can slow the LLM response.

Fix: Reduce context budget in Settings. Archive old sessions. Use a faster model for simple tasks.

High memory usage

Large embeddings or many documents in memory.

Fix: Run the cleanup route. Archive old documents. Restart the app to free leaked memory.
