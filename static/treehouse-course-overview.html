Modules

1. System Orientation

Start here. Understand what OpenClank is, how it starts, and where things live.

  • What is OpenClank? — Local-first AI assistant, multi-provider, persistent
  • Data Directories & Startup — ~/.odysseus/, settings, health checks
  • Settings & Configuration — Settings model, providers, env vars
  • Logs, Shutdown & Backup — Graceful shutdown, backup/restore, diagnostics

2. Models, Auth & Providers

Configure LLM providers, manage API keys, and understand model routing.

  • Provider Architecture — OpenAI, Anthropic, Google, Local, MiMo dispatch
  • API Key Management — Secure storage, atomic save, env var overrides

3. Agent, Tools & MCP

Master the Agent loop, MCP servers, built-in tools, permissions, and sub-agents.

  • The Agent Loop — Execution engine, context budget, compaction
  • Built-in Tools — File, shell, knowledge, orchestration tools
  • MCP Server Integration — External tool/resource providers, security
  • Skills & Permissions — Skill files, permission model, sensitive ops

4. Memory, Trust & Review

Capture, review, promote, and recall memory. Understand trust boundaries.

  • Memory Capture — Session, project, global memory layers
  • Trust & Memory Review — Checkpoint-derived, user-stated, auto-discovered

5. Copal Views

Work with Notes, Wiki, Events, Timeline, Bases, Tasks, and Calendar.

  • Notes & Wiki — Database-native documents, versioning, full-text search
  • Events & Timeline — Immutable event ledger, projections, audit trail
  • Bases, Tasks & Calendar — Structured data, task management, CalDAV sync

6. Security & Privacy

Understand owner scope, credential handling, backup/restore, and the threat model.

  • Owner Scope & Access Control — Profiles, roles, workspace isolation
  • Credential Handling & Threat Model — Secrets safety, threat model, backup

7. Capstone Project

Demonstrate end-to-end mastery with a hands-on project.

  • Inspect the App — Startup, health, provider, tool verification
  • Build Copal Knowledge — Documents, Bases, Timeline verification
  • Recovery & Restart — Backup, restart, integrity check

How It Works

TreeHouse tracks your progress through durable events, not browser-only counters. Every activity completion, submission, and grade emits an immutable event. Your points, badges, quests, and streak are computed from these events on every read.

Module-order gating ensures you complete items in sequence. Skill prerequisites gate access to advanced content. The instructor can create, edit, publish, and delete content. Learners enroll, complete, submit, and track their own progress.
